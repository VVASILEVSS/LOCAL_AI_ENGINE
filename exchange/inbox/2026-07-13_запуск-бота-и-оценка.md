## ЗАДАЧА 1: Запустить бота

Запусти `python main.py` на машине оператора. 

### Перед запуском проверь .env:
```
TOKEN=... (Telegram бот токен)
MY_CHAT_ID=... (chat ID оператора)
LOCAL_AI_ENDPOINT=http://localhost:1234/v1/chat/completions
MODEL_NAME=qwen_qwen2.5-vl-7b-instruct
```

Локальная LLM (LM Studio) должна быть запущена с моделью Qwen Qwen2.5 VL 7B Instruct.

### Что ожидать:
1. При старте — логи: `🕒 Планировщик запущен (интервал: 60 мин, инструменты: ['BTCUSDT', 'XAUTUSDT'], ТФ: ['1D', '4h', '1h', '15m'])`
2. Первый цикл начнётся сразу (APScheduler `misfire_grace_time`)
3. Для каждого символа: 4 графика → LLM вызов (2 раза для self-consistency) → enforce_risk_rules → TG сообщение
4. Примерное время первого цикла: 30-90 сек

### Что смотреть в логах:
- `LLM OK: model=..., tokens=..., attempt=1` — LLM отвечает
- `signal_log saved: id=N BTCUSDT ...` — backtest записывает
- `backtest: checked N forecasts` — проверка старых прогнозов
- Любые `ERROR` или `WARNING` — собрать в отчёт

### Известные особенности:
- **XAUTUSDT** (золото PAX Gold) — spot-only на Binance. `fetch_binance_metrics()` может вернуть "Данные временно недоступны" для золота (funding/OI не существуют для spot). Это ОК, бот продолжит работу.
- **Multi-symbol** — CoinGecko запросы (BTC dominance, Fear&Greed, ETH/BTC) могут быть rate-limited. Контекст будет "данные недоступны" — это ОК.
- **Первый анализ** — backtest покажет "статистика формируется (прогнозы накопляются)" — нормально, нужно ~4 часа для первых checked прогнозов.

## ЗАДАЧА 3: Проверить качество LLM ответа

Когда получишь ответ от бота для BTCUSDT — проверь:

1. **Все ли поля на месте:**
   - Цена (💰)
   - Prev trend (🧭)
   - HTF/LTF структура
   - Волновая фаза
   - ABC риск
   - Зоны по ТФ (4 таймфрейма)
   - Confluence levels
   - Risk management (primary: SL, TP1, TP2, TP3, RR)
   - Risk management (alternative: SL, TP1, TP2, TP3, RR)
   - signal_status
   - confidence

2. **Логическая согласность:**
   - Если signal_status = no_signal → risk_management должен быть пустой (SL=N/Д, TP=N/Д)
   - Если signal_status = aggressive_breakout/retest → должны быть заполнены primary SL/TP
   - SL для Long < цена, TP для Long > цена
   - RR > 1.0 (если risk блок не пустой)
   - tf_zones: D1 диапазон шире чем 15M

3. **Формат:**
   - Нет "[]", "{}", "None" в тексте (должно быть "Н/Д" или скрыто)
   - Текст на русском
   - Нет китайских символов
