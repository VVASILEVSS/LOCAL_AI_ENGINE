## Контекст

Все P0–P3 фичи закоммичены. Бот должен проходить полный цикл:
`scheduler → metrics → zigzag → heatmap → state → LLM → enforce_risk_rules → normalize → backtest → TG`

Нужно проверить **каждый этап** цикла на корректность данных, отсутствие падений и логическую связность.
Бот запускается на **реальной машине оператора** (не в этом sandbox), поэтому проверки — **статический анализ кода + синтетические тесты** (если возможно запустить).
