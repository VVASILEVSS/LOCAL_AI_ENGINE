### 3. MZZ4 параметры

**Одинаковые (12,5,3) для всех TF правильный подход.** Почему:
- Depth=12 — минимум пивотов для признания свинга
- Deviation=5%, Backstep=3 — чувствительность одинаковая
- Разный lookback (M15=5000, D1=150) — нормально, это просто сколько истории загружаем
- Альтернатива (14,28,102 / 1,8,21 / 3,5,12) — для MT4 где старшие TF меньше свечей имеют. У нас 200 свечей на каждый TF, параметры одинаковые логичнее

Наш `_find_real_pivots` в benchmark_zigzag.py использует `min_span=0.001` (0.1%) — это Deviation. Depth = неявно через min_pivots в accumulation. Работает аналогично.

### 4. ZigZag benchmark — контаминирует?

**Нет.** `structure.py` → `analyze_tf_structure` вызывается **отдельно для каждого TF** с **своими** swing_points. Parent zone передаётся только как constraint (soft clamp 10%), не как substitute. Поэтому per-TF зоны независимые.

ETH пример из твоего письма это подтверждает:
- ZigZag H4: [1750-1896] ✅
- ZigZag H1: [1779-1848] ✅
- ZigZag M15: [1751-1896] ✅

Все разные. Проблема исключительно в LLM output.

### 5. Confluence фильтр — нужен?

**При варианте E — нет.** Zones из ZigZag авторитетные, confluence проверка излишняя.

**При B+D (переходный) — да, как sanity check.** Но с поправкой: если зона имеет валидный BOS (bos_age < N свечей) → confluence не заменяет. Это твой вариант C, который ты уже частично реализовал.
