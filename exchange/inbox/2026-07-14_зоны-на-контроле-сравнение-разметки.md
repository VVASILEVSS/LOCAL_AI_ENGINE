### Расхождения:

**1. D1 low: 57800 (Binance) vs ~56804 (amarkets)**
Разница ~1000$. Это разница данных, не код. На Binance D1 low за 300 свечей = 57800 (1 июля). У пользователя на amarkets = ~56804. Нормально для разных брокеров.

**2. D1 zone [57800-64700] vs пользователь [56804-81304]**
Бот: zone = **post-BOS range** (текущая структура после BOS @ 62272). Высокий = 64700 (июльский high).
Пользователь: zone = **видимый range на графике** (весь тренд от мая). Высокий = ~81304 (майский rallу, prev_structure).

Это **концептуальное различие**, не баг. По нашему дизайну zone = curr_structure. Пользователь видит зону шире — включает prev_structure highs.

**3. Общий low не работает**
Пользователь ожидает один low (~56804) для всех ТФ. Бот даёт разные lows:
- D1: 57800, H4: 61825, H1: 62500, M15: 62534

Причина: каждый ТФ видит разные данные (разные windows) + top-down clamp не выравнивает lows, только ограничивает.

**4. H4/H1 highs — улучшились с depth=3**
H1 сузился с 4.1% до 2.1%. Это ближе к тому что ожидает пользователь.

## 2. Что нужно мониторить

### Зона D1 — слишком узкая относительно графика

Бот D1: [57800-64700] = текущая пост-BOS структура.
Пользователь видит: [56804-81304] = весь нисходящий тренд.

Для LLM сигнала важны ОБА range:
- **Текущая зона** (curr_structure) — где цена сейчас
- **Проблемные уровни** из prev_structure — куда цена может пойти

Сейчас narrative показывает prev_structure highs, но они не попадают в `tf_zones` и не видны LLM как уровни. Это OK для концепции B (LLM сам рисует), но проблема для концепции C.

### Top-down convergence lows

Идеальный паттерн (как у пользователя): все ТФ сходятся на одном low. В реальности bot даёт расхождение 57800→61825→62500→62534. Это потому что:
1. H4 видит 50 дней данных, H1 — 80 часов, M15 — 50 свечей
2. Базовый low разный из-за разных окон
3. Soft clamp не подтягивает lows вверх (только ограничивает снизу)

Возможный фикс: если child low > parent low — оставить child. Но если child low СЛИШКОМ выше parent low (>X%) — может стоит использовать parent low? Это опасно, может исказить структуру. Не трогать пока.

### M15 корректный после T1-T5 merge

Раньше M15 high = H1 high (наследование). Сейчас M15: [62534-63854] — из собственных пивотов, внутри H1. OK.
