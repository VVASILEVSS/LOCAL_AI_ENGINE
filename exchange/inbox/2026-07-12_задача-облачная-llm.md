## Твоя задача

### 1. Перенести верифицированное решение из AI-Analyzer-Plan в LOCAL_AI_ENGINE

Ты уже сделал это в AI-Analyzer-Plan (файлы: `config.py`, `ollama_service.py`, `.env.example`). Теперь нужно:

1.1. **Создай `core/ollama_service.py`** в LOCAL_AI_ENGINE — единый модуль для вызова LLM.
   - Функция `generate(prompt, **kwargs)` — универсальный вызов
   - Поддержка **двух режимов**:
     - **Cloud** (по умолчанию) — httpx → `{ENDPOINT}/chat/completions` с API-ключом в заголовке
     - **Local** (backward compat) — то же самое но без ключа, для тех кто хочет локальную LLM Studio
   - Retry с backoff (3 попытки, exponential)
   - Таймаут 30 сек
   - Возврат: `{"content": "...", "model": "...", "usage": {...}}` или raise
   - **НЕ использовать** библиотеку `ollama` — только httpx (ты уже так сделал)

1.2. **Обнови `core/config.py`** — добавить:
   ```python
   LLM_API_KEY = os.getenv("LLM_API_KEY", "")  # пустой = local mode
   LLM_MODE = "cloud" if LLM_API_KEY else "local"
   ```
   Оставить `LOCAL_AI_ENDPOINT` и `MODEL_NAME` как есть (уже через env vars).

1.3. **Обнови `.env.example`** — добавить `LLM_API_KEY=your_key_here` и комментарий что пустой = local mode.

### 2. Интегрировать ollama_service.py в ollama_client.py

2.1. **В `core/ollama_client.py`** — заменить прямой httpx-вызов LLM на вызов через `ollama_service.generate()`.

   Сейчас в `ollama_client.py` (~строки 100-200) есть функция которая делает httpx-post напрямую к LM Studio. Заменить на:
   ```python
   from core.ollama_service import generate as llm_generate
   ```
   И использовать `llm_generate(prompt, model=MODEL_NAME, ...)` вместо прямого httpx.

2.2. **Сохранить всю логику парсинга JSON** в `ollama_client.py` — она отличная, не трогать.
   Меняется только транспортный слой (как отправляется запрос к LLM).

2.3. **Обработка ошибок** — если `llm_generate` упал (таймаут, 500, нет ключа):
   - Записать в лог
   - Отправить в TG: "Ошибка LLM: {reason}. Попытка {n}/3"
   - Если все 3 попытки failed — "Анализ недоступен. Проверьте LLM_ENDPOINT и API_KEY."
