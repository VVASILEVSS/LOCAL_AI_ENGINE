## Суть

Оператор хочет запускать бота через веб-интерфейс, а не `python main.py` в консоли. Нужен простой веб-дашборд который:
1. Запускает бота **в фоне** (scheduler + TG bot) при открытии страницы
2. Либо по расписанию — **ежедневно в 09:00** (Asia/Qyzylorda, UTC+5)
3. Показывает статистику из `forecasts.db` в реальном времени

---

## Архитектура

```
web_dashboard.py  (Flask/FastAPI — ОДИН файл)
├── GET /          → HTML дашборд (статистика)
├── GET /api/stats → JSON (для автообновления)
├── POST /api/start  → запустить бота в фоне
├── POST /api/stop   → остановить бота
└── Фоновый поток: asyncio loop с Telegram ботом + scheduler
```

**Почему ОДИН файл:**
- Минимум зависимостей (Flask уже есть косвенно через aiogram/aiohttp)
- Нет фронтенд-билдера — чистый HTML/CSS/JS inline
- Оператор запускает: `python web_dashboard.py` — и открывает `http://localhost:5000`

## ТЕХНИЧЕСКИЕ ТРЕБОВАНИЯ

### 1. Фоновый запуск бота

```python
import threading
import asyncio

def run_bot():
    """Запуск Telegram бота + scheduler в отдельном потоке."""
    asyncio.run(main())  # из main.py

# При старте веб-сервера:
bot_thread = threading.Thread(target=run_bot, daemon=True)
bot_thread.start()
```

- `daemon=True` — поток умрёт при закрытии веб-сервера
- Бот стартует **сразу при запуске `web_dashboard.py`** (не ждёт 9:00)
- Первый цикл анализа начнётся по расписанию scheduler (сейчас interval=60 мин)

### 2. Ежедневный автозапуск в 09:00

Два варианта (выбери лучший):

**Вариант A — через APScheduler (рекомендуемый):**
В `start_scheduler()` добавить cron-триггер: запуск `run_hourly_analysis()` каждый день в 09:00 UTC+5. Но scheduler уже есть с interval — нужно **добавить** вторую job, не заменить.

```python
from apscheduler.triggers.cron import CronTrigger

scheduler.add_job(
    run_hourly_analysis,
    CronTrigger(hour=4, minute=0, timezone="UTC"),  # 09:00 UTC+5 = 04:00 UTC
    args=[bot],
    id="daily_analysis",
    replace_existing=True,
    max_instances=1,
    coalesce=True,
)
```

**Вариант Б — через systemd/cron на машине:**
Создать файл `scripts/start-bot.sh` и инструкцию для crontab. Менее надёжно, зависит от ОС.

**Реализуй Вариант А** (внутри кода, без зависимости от ОС).

### 3. Дашборд — что показывать

Читаем из `forecasts.db` (таблицы `forecasts` и `signal_log`):

#### Верхняя панель (статус)
- Статус бота: 🟢 Running / 🔴 Stopped
- Время последнего анализа
- LLM модель (из config)
- PROMPT_VARIANT (A или B)
- Интервал (мин)

#### Блок "Точность" (из signal_log)
- Всего прогнозов: N
- Проверено: N
- Accuracy: X% (wins/checked)
- TP1 hit rate: X%
- SL hit rate: X%
- No hit: X%
- Avg RR planned: X
- Avg RR realised: X

#### Блок "A/B тест" (из signal_log)
- Variant A: X/Y (Z%)
- Variant B: X/Y (Z%)

#### Блок "Последние 10 сигналов" (таблица)
| Время | Символ | Signal | Direction | Entry | SL | TP1 | Outcome | RR |
|-------|--------|--------|-----------|-------|----|-----|---------|-----|

#### Блок "По типам сигналов" (из signal_log GROUP BY)
| signal_status | Кол-во | TP hit % |
|---------------|--------|----------|

#### Автообновление
- `/api/stats` возвращает JSON со всей статистикой
- JS на странице запрашивает каждые 60 секунд
- Если бот Running — показывать "последнее обновление: ..."

### 4. HTML/CSS

- Тёмная тема (фон #1a1a2e или подобный)
- Шрифт: monospace для цифр, sans-serif для текста
- Адаптивный (работает на телефоне)
- Минимум внешних зависимостей — только inline CSS

## ПРИМЕЧАНИЯ

1. SQLite — потокобезопасен для чтения. Пишет только бот (один поток). Дашборд только читает.
2. Бот и веб-сервер в одном процессе — проще для оператора (одна команда запуска)
3. Не трогай существующие файлы (main.py, scheduler.py) — только читай из них
4. Если Flask не подходит — можно на `aiohttp.web` (уже есть в зависимостях через aiogram)
5. Порт 5000 по умолчанию, но сделай configurable через `WEB_PORT` env var
6. Коммит с префиксом `[LOCAL_AI_ENGINE]`
