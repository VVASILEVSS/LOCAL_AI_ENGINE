## Баг 2: split_structure — абсолютный max вместо последней суб-структуры

Выбрал **вариант A** (последний swing каждого типа перед BOS).

### Почему A, не C/D

**A чисто структурно:** BOS сломал конкретный range — range между последним swing high и последним swing low перед моментом слома. LH2 (82380) = уровень который держал цену перед BOS. LH1 (97924) = уровень из другой суб-структуры, цена уже откатилась от него к LH2.

**Почему не C (last N pivots):** N = гиперпараметр. При 3 пивотах на H4 и 15 на D1 — одно N не работает для всех. А при разных N по ТФ — мы в hyperspace.

**Почему не D (structural window для prev):** K = ещё один гиперпараметр. Плюс: если данные уже обрезаны structural window (50 свечей для 15m), двойное окно = двойная обрезка.

**Крайний случай A:** если последний high = мелкий pullback (например откат на 0.5% перед продолжением). Но это правильно — именно этот микро-range был сломан BOS. Если нужен более широкий контекст — это задача narrative (показать LH1, LH2, BOS), а не zone.

### Изменение

```python
# Было:
prev_h = max(p["price"] for p in prev_pivots if p["type"] == "high")
prev_l = min(p["price"] for p in prev_pivots if p["type"] == "low")

# Стало:

```

### Мой вариант реализации

**Модуль:** `core/trend_lines.py` (отдельный, не в structure.py). Причина: structure.py уже 650+ строк, trend lines = отдельная концепция (наклонные vs горизонтальные).

**T8: detect_trend_lines()**

```python
def detect_trend_lines(
    swing_points: List[Dict[str, Any]],
    direction: str,  # "bullish" (HL series) or "bearish" (LH series)
    min_points: int = 3,
) -> Optional[TrendLine]:
```

**Сколько пивотов:** **min_points=3, без upper limit.** Берём все последовательные LH (bearish) или HL (bullish) с конца. Останавливаемся когда последовательность прерывается (bearish: появился higher high; bullish: появился lower low).

Почему 3 minimum:
- 2 точки = линия всегда существует (тривиально)
- 3 точки = линия имеет смысл (тренд подтверждён)
- Больше = лучше, но если есть только 3 — этого достаточно

Почему без upper limit:
- Если 5 LH подряд — тренд сильный, все 5 точек полезны для r²
- r² автоматически отфильтрует шум

**r² — через numpy:**

```python
import numpy as np

xs = np.array([p["index"] for p in points])
ys = np.array([p["price"] for p in points])
slope, intercept = np.polyfit(xs, ys, 1)
y_pred = slope * xs + intercept
ss_res = np.sum((ys - y_pred) ** 2)
ss_tot = np.sum((ys - np.mean(ys)) ** 2)
r_squared = 1 - ss_res / ss_tot if ss_tot > 0 else 0.0
```

numpy.polyfit = O(n), r² = O(n). Для 5-15 пивотов — мгновенно.

**Slope — % per candle (log returns):**

```python
# Не raw % — log returns для нормализации
slope_pct = slope / np.mean(ys) * 100  # % per candle (raw)
# Или log:
log_slope = (np.log(ys[-1]) - np.log(ys[0])) / (xs[-1] - xs[0]) * 100  # % per candle (log)
```

Я за **log returns** — они симметричны (рост 10% = падение -10% в log), в отличие от raw % (рост 10% → 110, падение 10% от 110 → 99, не 100).

**angle_class:** adapted per TF (у D1 0.1%/day = умеренный, у 5m 0.1%/candle = очень крутой):

```python
# Нормализуем к дневному эквиваленту
candles_per_day = {"1d": 1, "4h": 6, "1h": 24, "15m": 96, "5m": 288}
daily_slope = log_slope * candles_per_day.get(tf, 24)

if abs(daily_slope) > 2.0:
    angle_class = "steep"
elif abs(daily_slope) > 0.5:
    angle_class = "moderate"
else:
    angle_class = "shallow"
```

**T9: check_trend_line_break()**

```python
def check_trend_line_break(
    current_price: float,
    line: TrendLine,
    closes: List[float],
    swing_points: List[Dict],
    lookback: int = 5,
) -> Dict[str, Any]:
```

**False breakout — не по lookback, по структуре:**

Твоё предложение: "возврат под линию за lookback свечей". Мой вариант: lookback как начальный фильтр + **структурное подтверждение**:

```python
# 1. Цена пересекла линию (close)
line_price_at_end = line.slope * len(closes) + line.intercept
crossed = (line.direction == "bearish" and closes[-1] > line_price_at_end) or \
          (line.direction == "bullish" and closes[-1] < line_price_at_end)

if not crossed:
    return {"broken": False, "break_type": "none"}

# 2. Проверяем lookback свечей — вернулась ли цена
recent = closes[-lookback:]
if line.direction == "bearish":
    # Для пробоя bearish линии — нужно чтобы цена оставалась выше
    still_above = all(c > line.slope * (len(closes) - lookback + i) + line.intercept
                     for i, c in enumerate(recent))
    if not still_above:
        return {"broken": True, "break_type": "false", ...}
else:
    still_below = all(c < line.slope * (len(closes) - lookback + i) + line.intercept
                     for i, c in enumerate(recent))
    if not still_below:
        return {"broken": True, "break_type": "false", ...}

# 3. Структурное подтверждение: следующий pivot обновляет экстремум?
# (для true breakout — нужен HH после пробоя bearish линии)
return {"broken": True, "break_type": "true", ...}
```

Но пункт 3 (структурное подтверждение) нельзя проверить мгновенно — нужно ждать следующий pivot. Поэтому:

- **Мгновенный вердикт:** `potential` (цена пересекла + lookback OK)
- **Подтверждённый:** `true` (после следующего pivot обновляет HH/LL)
- **Опровергнутый:** `false` (возврат за lookback)

**5 открытых вопросов — мои ответы:**

**Q1: Сколько пивотов для линии?**
→ min 3, без upper limit. Берём последовательные LH/HL с конца до прерывания.

**Q2: r² через numpy?**
→ Да. `np.polyfit(xs, ys, 1)` + ручной r². O(n), нет зависимостей кроме numpy (уже есть).

**Q3: False breakout — lookback?**
→ lookback=5 свечей как фильтр + `potential/true/false` тройка. True подтверждается структурно (новый HH/LL), не временным окном.

**Q4: Угол — % per candle vs log vs ATR-normalized?**
→ **Log returns, нормализованные к дневному эквиваленту.** ATR-normalized — интересная идея (angle = slope/ATR), но ATR = волатильность, не масштаб. Если ATR вырос в 2x, угол не должен меняться в 2x. Log returns = правильная метрика угла.

**Q5: Модуль?**
→ `core/trend_lines.py`. Structure.py = 650+ строк, trend lines = отдельная концепция.
