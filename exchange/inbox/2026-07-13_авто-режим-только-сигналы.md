# Задача: Авто-режим — отправлять только при подтверждённом сигнале

**От:** Super Z
**Кому:** Hermes
**Дата:** 2026-07-13
**Приоритет:** Средний

---

## Суть

Сейчас `scheduler.py:427` — `send_to_tg = True` всегда. Нужно добавить режим:

- **Ручной (по умолчанию)** — отправлять каждый анализ (текущее поведение)
- **Авто** — отправлять в TG ТОЛЬКО когда есть подтверждённый сигнал

---

## Что менять

### 1. `core/scheduler.py` — строка 427

Заменить:
```python
send_to_tg = True
```

На:
```python
# Авто-режим: отправлять только при сигнале
auto_mode = get_setting("auto_mode", False)
active_signals = ("aggressive_breakout", "retest", "reversal", "false_breakout")
send_to_tg = (not auto_mode) or (status in active_signals)
```

### 2. `core/handlers.py` — добавить команду `/auto`

По аналогии с существующим `/filter` (строки ~390-396):

```python
@router.message(Command("auto"))
async def cmd_auto(message: Message):
    """Тогл авто-режима: только сигналы в TG."""
    state = not get_setting("auto_mode", False)
    set_setting("auto_mode", state)
    status_text = "ON (только сигналы)" if state else "OFF (все анализы)"
    await message.answer(f"🔇 Авто-режим: {status_text}")
```

### 3. `core/db.py` — добавить дефолт

В `init_all_tables()`, в блоке INSERT OR IGNORE для settings, добавить:
```python
c.execute('''INSERT OR IGNORE INTO settings (key, value) VALUES ('auto_mode', 'false')''')
```

### 4. `core/scheduler.py:441-452` — save_forecast

Сейчас `save_forecast()` вызывается ТОЛЬКО при `aggressive_breakout` и `retest`. Это правильно, не трогать.

Но в авто-режиме предупреждения об уровне (warning_msg) тоже не нужно отправлять при no_signal. Логика уже правильная — warning_msg добавляется к msg_text, а msg_text отправляется только если `send_to_tg = True`.

---

## Проверка

- `auto_mode = false` → каждые 60 мин приходит полный анализ (как сейчас)
- `auto_mode = true` → приходит ТОЛЬКО при aggressive_breakout/retest/reversal/false_breakout
- Ошибки анализа (parsed.get("error")) — ВСЕГДА отправлять, независимо от режима
- Backtest (`save_signal_log`) — ВСЕГДА записывать, независимо от режима
- `/auto` в боте — тоглит режим
