## Суть задачи

Реализовать механизм self-consistency для LLM-анализа: **2 прогона** вместо 1, с голосованием по `signal_status`. Это уменьшит случайные галлюцинации модели.

## Технические требования

### 1. Функция `analyze_multi_images` в `core/ollama_client.py`

Сейчас функция вызывает LLM один раз. Нужно:

```python
# Псевдокод:
results = []
for run_idx in range(2):
    raw = await generate(messages=messages, temperature=0.15 + run_idx * 0.1, ...)
    parsed = parse_llm_json(raw)
    if not parsed.get("error"):
        results.append(parsed)

if not results:
    return {"error": True, "message": "Both runs failed"}

# Голосование по signal_status
from collections import Counter
signals = [r.get("signal_status", "no_signal") for r in results]
winner_signal = Counter(signals).most_common(1)[0][0]

# Берём результат с мажоритарным signal_status
final = next(r for r in results if r.get("signal_status") == winner_signal)
```

### 2. Ограничения

- **Только 2 прогона** (не 3 — чтобы не удваивать время: 25с → 50с приемлемо, 75с нет)
- **Temperature**: первый прогон = 0.15, второй = 0.25 (небольшая вариация для разнообразия)
- **Timeout**: если первый прогон занял > 40с — второй пропускать (защита от таймаута)
- **Логирование**: `logger.info("Self-consistency: run %d/%d, signal=%s", run_idx+1, 2, signal)`
- **Если оба прогона дали разный signal_status** — логировать `logger.warning("Self-consistency disagreement: %s vs %s", s1, s2)` и брать первый (более детерминированный, temperature=0.15)

### 3. Формат результата

Добавить в итоговый dict:
```python
final["_consistency"] = {
    "runs": len(results),
    "signals": signals,
    "agreed": len(set(signals)) == 1,
}
```
