### 1. Вариант A: ZigZag structure fallback (ollama_client.py)

**Что:** После `_validate_min_span` и `_enforce_zone_uniqueness` — если зона удалена, подставляю зону из ZigZag `curr_structure` (реальные пивоты + BOS).

**Где:** `enforce_risk_rules()`, блок 2d (после 2c min-span).

**Логика:**
```
для каждого ТФ (5M, 15M, 1H, 4H, 1D):
  если зона есть в tf_zones → skip
  если зона удалена → взять из zigzag_context.timeframes[tf].structure.curr_structure
  fallback: если structure нет → взять raw upper/lower из zigzag
  пометить source = "zigzag_structure_fallback"
```

**Почему это работает:**
- ZigZag зона = структурный range после BOS (из реальных пивотов)
- LLM зона = микроканал последних свечей (визуальное сжатие)
- Fallback даёт реальный range, не микроканал
- Если ZigZag тоже даёт узкую зону (реальное сжатие) → min-span её тоже удалит, и fallback не сработает (upper <= lower) → зона останется пустой. Это правильно — реальное сжатие = нет зоны.

**Ответ на твой вопрос про sticking:**
Sticking detection на XAUT не сработал потому что:
1. `_enforce_zone_uniqueness` проверяет пары parent→child (D1→H4, H4→H1, H1→15M)
2. Но на XAUT `15m lower = 1h lower = 4h lower` — это 3-уровневое sticking, а проверка только 2-уровневая
3. H1→15M: если H1 уже расширен (expand_pct=0.0075), то после расширения lower/upper отличаются от 15m → sticking не детектируется
4. Теперь с fallback это не критично — если 15m удалена, подставится ZigZag zone

### 2. Вариант D: Структурное окно (benchmark_zigzag.py)

**Что:** Для младших ТФ обрезаю данные до последних N свечей перед поиском пивотов.

**Окна:**
- 5m: последние 50 свечей
- 15m: последние 50 свечей
- 1h: последние 80 свечей
- 4h: все данные (200-300) — мало пивотов, не нужно ограничивать
- 1d: все данные (200-250) — аналогично

**Эффект:**
- 15m ETH: вместо 200 свечей с range 1748-1848 (5.59%) → 50 свечей с ~2 структурными движениями
- BOS будет обнаружен в контексте этих 2 движений
- Zone = range текущей структуры после BOS (структурный, не микроканал)

**Технические детали:**
- Пивоты ищутся в обрезанном окне (highs[-50:], lows[-50:])
- Closes для BOS тоже обрезанные — индексы совпадают
- Индексы пивотов в локальных координатах (0..49) — не сдвигаются
- total_candles для structure analysis = window (не len(df))

### 3. Промпт: правило 8c

Добавил в user prompt:

> **8c.** Зона для каждого ТФ = range от последнего слома структуры (BOS) до текущего момента. НЕ ставь зону = микроканал последних 5-10 свечей. Если видишь сжатие — расширь зону до ближайших структурных high/low.

Это прямая инструкция LLM не копировать микроканал. Работает в связке с structure narrative (который показывает BOS + prev/curr zone).

### 4. Твой фикс detect_bos() (3f45974)

Подтверждаю — твой фикс корректный. Классический SMC BOS: close пробивает **предыдущий** swing level, не последний. Моя оригинальная логика была слишком узкой — цена почти всегда между last SH и last SL, поэтому 8/8 None.

Твой фикс:
- Идёт с конца по swing_highs (кроме последнего)
- Ищет close > этого high + есть swing_low после → bullish BOS
- Аналогично для bearish
- Берёт более свежий из bullish/bearish BOS

Также увидел расширение `curr_pivots` в `split_structure` — если после BOS < 2 типов (нет high после bullish BOS) → добавляем пивоты до BOS. Это фиксит коллапс зоны. Хорошее решение.
