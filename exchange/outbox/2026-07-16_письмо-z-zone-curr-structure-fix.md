## Что произошло

User дал концепцию машины состояний зоны после пробоя (по методологии Вячеслава Возного, YouTube канал `@Vyacheslav_Voznyy` — https://www.youtube.com/@Vyacheslav_Voznyy). Я изучил 4 видео канала + сегодняшний эфир «РОСТ ЭФИРА И АЛЬТКОИНОВ» и применил фиксы в `core/structure.py`.

**Ключевая концепция (Возный):**
- Зона = накопление = структурный диапазон **последней суб-структуры после BOS**
- Зона ≠ union(prev + curr) — это было главным багом
- BOS = close телом за структурным уровнем (не фитиль)
- Агрессивный сигнал = сам пробой границы (минимальный объём, SL по краю)
- Консервативный сигнал = после ретеста пробитой зоны + новый BOS своего свинга (больший объём, безубыток, множественные TP)
- После пробоя зона **замораживается** — не перестраивается пока не сформируется суб-структура (≥1 swing H + ≥1 swing L, чередование = тренд)
- Ложный пробой = снятие ликвидности фитилем → возврат в зону → ждём пробой противоположной границы

## Изменения в `core/structure.py` (НЕ закоммичено, на main)

### Фикс 1: Зона = curr_structure ТОЛЬКО (убрал union)
**Было** (lines ~408-417):
```python
zone_high = curr_struct.high
zone_low = curr_struct.low
if prev_struct:
    zone_high = max(zone_high, prev_struct.high)   # расширяло ВВЕРХ
    zone_low = min(zone_low, prev_struct.low)       # расширяло ВНИЗ
```
**Стало:** убрал блок `if prev_struct`. Зона = `curr_struct.high/low` напрямую. `prev_structure` остаётся в dataclass как контекст, но НЕ расширяет зону.

### Фикс 2: Убрал фабрикацию curr_struct
**Было** (lines ~293-305): если после BOS <1 swing каждого типа → добавлял пивоты ДО BOS (фабрикует curr).
**Стало:** не расширяет назад. Если нет swing H или L после BOS → `curr_h`/`curr_l` = `None`.

### Фикс 3: BOS.price = граница новой зоны
Если после BOS нет swing H → `curr_high = bos.price` (пробитый swing low → resistance).
Если нет swing L → `curr_low = bos.price`.
Логика: BOS пробивает swing → этот уровень становится границей новой зоны.

### Фикс 4: current_price НЕ в границах зоны
Убрал `h = max(curr_h, current_price)`. Цена — не граница зоны. Цена внутри зоны = накопление.

### Фикс 5: Zone breakout detection
**Было:** `prev_close >= zone_low and last_close < zone_low` (требовал переход изнутри наружу).
**Стало:** `last_close < zone_low` = пробой (факт close за границей, независимо от предыдущей свечи).
Добавил поля `zone_breakout_up: bool` и `zone_breakout_down: bool` в `StructureAnalysis` dataclass.

### Фикс 6: Parent constraint — не клампить если parent пробит
В `analyze_topdown`: если parent zone инвертирована (low > high) ИЛИ parent `zone_breakout_up/down=True` → **не передаём parent_zone** ребёнку. Логика: если старший ТФ пробит → его zone больше не constraint для младшего ТФ.
