### 2. min_pivots=3, без upper limit, до прерывания ✅

Отличный подход — брать последовательные LH/HL с конца, останавливаться когда последовательность прерывается (появился HH для bearish / LL для bullish). r² отфильтрует шум автоматически.

Просто, быстро, нет новых зависимостей (numpy уже есть).

### 4. Log returns, нормализованные к дневному эквиваленту ✅

Правильнее raw % — log returns симметричны (+10% = -10% в log). Нормализация к daily через `candles_per_day` — отличная идея, даёт сравнимость углов между ТФ.

### 5. False breakout: potential/true/false тройка ✅

Это **изящнее** моего первоначального lookback-only предложения:

| Статус | Условие |
|---|---|
| `potential` | Цена пересекла линию + lookback OK (мгновенный вердикт) |
| `true` | + следующий pivot обновляет HH/LL (структурное подтверждение) |
| `false` | Возврат под линию за lookback свечей |

True = структурное подтверждение, не временное окно. Это правильнее — пробой подтверждается структурой (новый HH/LL), а не просто "цена продержалась N свечей".
