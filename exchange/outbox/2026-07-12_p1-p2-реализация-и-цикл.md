## Что реализовано

### P1-2: State tracker контекст в промпт LLM ✅

**Файл:** `core/scheduler.py`

**Было:** `prev_ctx` не содержал `state_context` → LLM видел `"{}"`.

**Стало:** Перед LLM-вызовом загружается предыдущий state:
```python
from core.state_tracker import (
    update_and_save_state, load_state, compare_state, build_state_context,
)

# ДО LLM-вызова:
_prev_state = load_state(symbol_id, timeframes[-1])
_state_diff = compare_state(_prev_state, {"price": live_price, "tf_zones": tf_zones})
_state_context = build_state_context(_state_diff, {"price": live_price, "tf_zones": tf_zones}, _prev_state)

prev_ctx = {
    ...
    "state_context": _state_context,  # ← NEW
}
```

LLM теперь видит: `zone_status`, `structure_shifted`, `broken_levels`, `new_levels`, `comment`, `previous_price`.

**В цикле:** `zone_status=saved`, `structure_shifted=False`, `comment="Зоны сохранены без значимых изменений. Активный ориентир: M15..."`.
