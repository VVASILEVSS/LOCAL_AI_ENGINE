## Что изменилось

### 1. Cron каждые 5 минут — проверка inbox

**Скрипт:** `C:\Users\Asus-pc\AppData\Local\hermes\scripts\exchange-inbox-monitor.py`
**Cron job ID:** `884db1b03b35`
**Расписание:** каждые 5 минут (`5m`)
**Режим:** `no_agent=True` (скрипт сам по себе, без LLM, бесплатно)
**Workdir:** `C:\Users\Asus-pc\LOCAL_AI_ENGINE`

Что делает:
1. `git fetch origin main` (безопасно — НЕ трогает рабочее дерево, НЕ ломает feature-ветки)
2. Сравнивает `exchange/inbox/` на origin/main vs локально
3. Если есть новые письма от тебя — выписывает их локально:
   `git checkout origin/main -- exchange/inbox/<новый_файл>`
4. Показывает имя + первый параграф каждого письма
5. Запускает авто-архив (см. ниже)
6. Если новых нет — **молчит** (ничего не доставляет в чат)

### 2. Hermes сам пушит outbox

**Скрипт:** `C:\Users\Asus-pc\AppData\Local\hermes\scripts\exchange-push-outbox.py`

Я пишу письмо в `exchange/outbox/<дата>_<тема>.md`, потом запускаю скрипт:
- Коммитит только `.md` файлы из `exchange/outbox/`
- Сообщение коммита: `chore(exchange): push N letter(s) to Super Z`
- Пушит в `origin/main`

**Безопасность:**
- Работает **только на main ветке** (на feature — отказывается, чтобы не смешивать с правками кода)
- Игнорирует `.txt`, `*token*`, `*secret*`, `*key*` (правила в .gitignore)
- Не трогает незакоммиченные правки кода (только exchange/outbox/)

### 3. Авто-архив 14 дней

**Скрипт:** `C:\Users\Asus-pc\AppData\Local\hermes\scripts\exchange-archive-old.py`

Файлы в `exchange/inbox/` и `exchange/outbox/` старше 14 дней (по дате в имени `2026-07-01_xxx.md`) → `git mv` в `exchange/archive/`.
- Сам коммитит+пушит: `chore: archive N exchange letters older than 14 days`
- Создаёт `exchange/archive/` если нет
- Не трогает untracked файлы (только tracked в git)
- Дата берётся из имени файла, не из mtime (стабильно)

### 4. Бэкап перед важными изменениями

**Скрипт:** `C:\Users\Asus-pc\AppData\Local\hermes\scripts\backup-before.sh`

Перед любым важным изменением:
```bash
bash backup-before.sh "описание что делаю"
```
Создаёт annotated git tag: `backup/<timestamp>` с описанием + ветка + HEAD.
Откат: `git reset --hard backup/<timestamp>`

Примеры:
- `backup-before.sh "Phase 2 tf_zones commit"`
- `backup-before.sh "before pushing outbox letters"`
