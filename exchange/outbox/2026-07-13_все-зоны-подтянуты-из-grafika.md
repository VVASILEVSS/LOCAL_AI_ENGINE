# Задание: ВСЕ зоны по ТФ подтянуты из графика — некорректные

## Проблема

После коммита 64aeb6e (D1 cap на fallback) пользователь сообщает: **ВСЕ зоны по ТФ некорректные, не только D1**.

## Корневая причина (установлена Super Z)

Проблема НЕ в D1 cap. Проблема в том, что **LLM почти не возвращает tf_zones** в JSON-ответе, и **почти все зоны идут из fallback `_fill_missing_tf_zones()`** — то есть это СЫРЫЕ технические экстремумы из `get_structural_extremums()` (auto_chart.py:192), а НЕ результат LLM-анализа.

### Что такое fallback-зона на самом деле

`all_metrics[tf]["zone"]` — это `get_structural_extremums(df, timeframe)`:
- Берёт 120 свечей
- Ищет pivot high/low с глубиной (depth=3..7 по ТФ)
- Возвращает `upper = max pivot high` и `lower = min pivot low` за 120 свечей
- Это МАКСИМАЛЬНЫЙ и МИНИМАЛЬНЫЙ экстремум за период — **не зона консолидации/диапазона**

**Это не то, что должно быть в "ЗОНЫ ПО ТФ"**. Пользователь ожидает видеть зоны LLM-анализа — диапазоны где цена консолидируется, а не технические экстремумы.

### Почему LLM не возвращает tf_zones

1. В `analyze_multi_images()` передаётся **ТОЛЬКО ОДНА картинка** (`b64_images[0]` = младший ТФ). Остальные графики не передаются в LLM (строка 2087-2089 ollama_client.py):
   ```python
   content_parts.append(
       {"type": "image_url", "image_url": {"url": f"data:image/jpeg;base64,{b64_images[0]}"}}
   )
   ```
   LLM видит только один график, а в тексте ему дают зоны для всех ТФ. GLM не может корректно вернуть зоны для графиков которые не видел.

2. Даже текстовый контекст с зонами не помогает — GLM часто просто не включает tf_zones в JSON-ответ.

## Что нужно сделать

### Приоритет 1: Передать ВСЕ графики в LLM

В `analyze_multi_images()` (ollama_client.py:2086-2089) сейчас добавляется только `b64_images[0]`. Нужно добавить ВСЕ картинки:

```python
for img_b64 in b64_images:
    content_parts.append(
        {"type": "image_url", "image_url": {"url": f"data:image/jpeg;base64,{img_b64}"}}
    )
```

**Осторожно**: GLM имеет лимит картинок. Если лимит превышен — можно передать 2-3 ключевых (HTF + LTF + средний), а не все 4-5.

### Приоритет 2: Если LLM всё равно не возвращает tf_zones — исправить fallback

Текущий fallback из `get_structural_extremums()` даёт мусорные зоны (max/min за 120 свечей). Варианты:

**Вариант A**: Если LLM не вернул tf_zones, взять их из `prev_analysis["tf_zones"]` (контекст который был передан в LLM — те же расчётные зоны но хотя бы в правильном формате) и прогнать через `enforce_risk_rules()` (там будет D1 cap и nesting).

**Вариант B**: Удалить fallback совсем. Если LLM не вернул зону для ТФ — не показывать её в "ЗОНЫ ПО ТФ". Лучше 2 правильные зоны от LLM чем 5 мусорных из графика.

**Вариант C (рекомендуемый Super Z)**: Комбинация:
- Передать все графики в LLM (приоритет 1) — это уменьшит количество missing зон
- Оставить fallback, но брать зоны из `prev_analysis["tf_zones"]` (они уже в правильном формате) и прогонять через `_validate_zone_nesting()` для D1 cap
- Если зоны нет ни в LLM ни в prev_analysis — НЕ подставлять мусор из `all_metrics[tf]["zone"]`
