### Q2: BOS close-only — ПРИНИМАЮ
Твой фикс `3f45974` уже close-only, работает. CHoCH (по wick) — потом, отдельная функция. Не сейчас.

### Q3: parent_zone параметр + clamp — ПРИНИМАЮ с soft clamp
Параметр в `analyze_tf_structure()` + soft constraint (10% tolerance, chain break) — твоё Предложение 2. Это критично: D1 false BOS не должен сломать всю цепочку. Принцип:
- zone_high > p_high * 1.10 → warning + parent_zone=None (chain break)
- zone_low < p_low * 0.90 → warning + parent_zone=None
- иначе → clamp `zone_high = min(zone_high, p_high)`

Цепочка прерывается, младший ТФ анализируется независимо. Это правильно.

### Q4: Накопление = 3+ пивота без обновления zone bounds — ПРИНИМАЮ
Алгоритм: последние `min_pivots` пивотов не пробивают zone_high (для high) / zone_low (для low) → accumulation. Адаптивный min_pivots по ТФ (D1=2, H4=3, H1=3, 15m=4, 5m=4) — разумно, младшим нужно больше подтверждений.

Поля в `StructureAnalysis`: `is_accumulation`, `accumulation_pivot_count`.

### Q5: Объём у уровня — ПРИНИМАЮ, но T6 (последний)
`volume_at_level()` — отдельная утилита. Радиус 1.5*ATR(14), 5 свечей lookback vs 20 avg. Сигналы: breakout (vol_ratio > 1.5), rejection (< 0.7), neutral.

**Не блокирует top-down.** T6 после T1-T5. Объём нужен только при подходе к уровню — не общий volume_ratio.

### Q6: _enforce_zone_uniqueness — ПРИНИМАЮ изменение
Логика: микроканал+sticking → удалить (fallback подставит). Структурная зона+sticking → confluence (не трогать). Убрать синтетическое раздувание parent (`expand_pct`).

### Q7: Цели = parent boundaries — ПРИНИМАЮ
`StructureAnalysis.targets: List[Dict]`. Формат: `{"level": float, "type": "parent_boundary"|"swing_level", "tf": str}`. Telegram-формат — моя задача.

### Q8: Двухфазный подход LLM — ПРИНИМАЮ
- **Фаза 1** (текущая): structure zones как narrative, LLM выдаёт свои upper/lower, post-hoc validation ловит ошибки
- **Фаза 2** (после top-down): `tf_zones_precomputed` в промпте, LLM корректирует только при явном расхождении
- **Фаза 3** (концепция C, долгосрочная): LLM не выдаёт upper/lower, только signal/entry/risk/narrative

Фаза 1→2 — минимальные изменения. Фаза 2→3 — требует согласования формата.
