## 3. Три предложения (выбери или предложи своё)

### Предложение A: clamp prev_struct к STRUCT_WINDOW (минимальный фикс)

```python
# core/structure.py — в split_structure(), после line 264
prev_pivots = [p for p in swing_points if p["index"] <= bos_idx]
# NEW: clamp prev_struct к окну STRUCT_WINDOW
window = _STRUCT_WINDOW.get(tf_norm)  # передать tf_norm в split_structure
if window:
    min_idx = max(0, total_candles - window)
    prev_pivots = [p for p in prev_pivots if p["index"] >= min_idx]
```
**Плюсы:** минимальный патч, не ломает пайплайн.
**Минусы:** если BOS в пределах окна — prev_struct может стать пустым → fallback.

### Предложение B: структурная зона = recent swing high/low (без BOS-зависимости)

```python
# Заменить zone = curr_struct на:
# zone = последние N значимых свингов в окне STRUCT_WINDOW
recent = [p for p in swing_points if p["index"] >= max(0, total_candles - window)]
if recent:
    zone_high = max(p["price"] for p in recent if p["type"] == "high")
    zone_low = min(p["price"] for p in recent if p["type"] == "low")
```
**Плюсы:** всегда структурная зона (недавние свинги), не зависит от BOS.
**Минусы:** теряется BOS-контекст (prev/curr фаза), надо отдельно трекать phase.

### Предложение C: гибрид — curr_struct + parent zone bounds (clip)

```python
# zone = curr_struct, но если curr span < 1% — расширить до parent zone
if curr_struct:
    span_pct = (curr_struct.high - curr_struct.low) / current_price * 100
    if span_pct < 1.0 and parent_zone:
        # Расширить до parent bounds, но не больше parent
        zone_high = max(curr_struct.high, parent_zone["high"] * 0.98)
        zone_low = min(curr_struct.low, parent_zone["low"] * 1.02)
```
**Плюсы:** сохраняет BOS-логику, расширяет микро-зоны через parent.
**Минусы:** parent bounds могут быть тоже микро (каскад).
