## Проблема

После удаления D1 cap ±10% (`8275d30`) — LLM возвращает **одну зону для D1/H4/H1**. Матрешка сломалась в обратную сторону.

            M15 [1766.76 ─ 1777]     ← единственный уникальный

Реальность: 1D last 3: H=1848, L=1748
            4H last 6: H=1848, L=1748
            1H last 24: H=1848, L=1748

- Бот upper (1830) ниже реального (1848) на $18

## Корень проблемы

1. Раньше cap ±10% делал D1 слишком широким (ZigZag copy → cap обрезал → 68491)
2. Теперь без cap LLM копирует **одну зону на 3 ТФ** (D1=H4=H1)
3. LLM ленится — возвращает одну зону вместо раздельных по ТФ

## Что нужно

**Золотая середина:**
- Не cap ±10% (сильно режет)
- Не без cap (LLM копирует одну зону)
- **Валидация вложенности** + **проверка уникальности** зон по ТФ

### Вариант A (минимальный, безопасный)

Добавить проверку: если D1==H4 (lower и upper совпадают) → попросить LLM сгенерить раздельно, или взять H4 из metrics (auto_chart) как fallback.

### Вариант B

В промпте явно сказать: "Для каждого ТФ — ОТДЕЛЬНАЯ зона. Не копируй D1 в H4/H1. H4 уже H4, H1 уже H1."

### Вариант C

В _fill_missing_tf_zones: если LLM вернул D1=H4=H1 → fallback на ZigZag/metrics зоны для H4 и H1 (они разные).
