### Правила работы

| Правило | Описание |
|---|---|
| **Старый код не трогать** | На `fix/zones-sticking` работаем только с `auto_chart.py`, `ollama_client.py`, `handlers.py`, `main.py` — тот код который уже работает в проде |
| **Новый код — в новый модуль** | `core/structure.py` живёт на `feature/top-down-structure`. Когда будет готов — сольём в main ОДНИМ PR |
| **Не мержить ветки друг в друга** | Каждая ветка мержится только в `main`. Между собой НЕ мержим — чтобы не тащить чужие изменения |
| **Письма в exchange/** | Все координационные сообщения — в `exchange/inbox/` (входящие) и `exchange/outbox/` (исходящие). Формат: `YYYY-MM-DD_от кого-тема.md` |

## 4. Промежуточные проверки — протокол

Каждая задача считается **завершённой** только когда:
1. Код закоммичен и запушен в правильную ветку
2. Написано письмо в `exchange/outbox/` с результатом (даже если «не работает — вот почему»)
3. Второй агент прочитал письмо и подтвердил: «вижу, OK» (письмо-подтверждение в `exchange/inbox/`)

**Формат письма-результата:**
```markdown
# Результат: [номер задачи]

**Статус:** DONE / PARTIAL / BLOCKED
**Ветка:** [имя ветки]
**Коммит:** [хеш]

## Что сделано
...

## Что не получилось
...

## Нужно от второго агента
...
```
