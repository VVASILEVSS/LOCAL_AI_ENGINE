### Проблемы ⚠️

**1. Post-hoc костыль, не root-cause fix.** Min-span удаляет узкие зоны → fallback подставляет VP/ZigZag. Но:

- VP может быть тоже узким (если реальное сжатие) → VP span < min_span → VP тоже удалить? → тогда что?
- ZigZag fallback (`_fill_missing_tf_zones`) даёт `upper=np.max(highs)`, `lower=np.min(lows)` за 200 свечей → span огромный (5-10%) → всегда проходит min-span, но это **не актуальная структура**, а raw экстремумы за полмесяца.

**Мин-span гарантирует что зона не микроканал, но НЕ гарантирует что зона = актуальная структура.** Удаление узкой зоны → подстановка raw max/min за 200 свечей = другая крайность (слишком широкая, не актуальная).

**2. `_validate_min_span` (ollama_client.py:~1040-1060) — мёртвый код.** Сначала `z["_min_span_rejected"] = True`, потом сразу `del tf_zones[tf_key]`. Mark бесполезен — зона уже удалена, mark некому читать. Не баг, но шум.

**3. Пороги могут убить легитимные узкие зоны.** Если рынок реально в сжатии ( ETH 15m ATR=0.5%), легитимная зона 0.6% будет удалена. Мин-span = 0.8% жёстковато для низковолатильных периодов.

## Главная проблема: модель НЕ видит структуру

Min-span validation — это фильтр на выходе. Но причина узких зон в том, что **LLM не видит структуру на входе**:

| Что LLM видит | Сейчас | Нужно |
|---------------|--------|-------|
| График | Свечи + SMA20 + BB + Fib | + ZigZag линия + BOS вертикаль + zone rect |
| ZigZag контекст | `mode=bearish pos=0.39 pivots=7` (абстракция) | "Пред: 1793→1748. BOS: 1793↑. Тек: 1748→1789" (narrative) |
| ZigZag данные | `upper=np.max(highs)` за 200 св (raw) | Актуальная структура после последнего BOS |
| Пивоты | ФЕЙК: каждая 5-я свеча alternating high/low | Реальные swing highs/lows |

### КРИТИЧЕСКОЕ ОТКРЫТИЕ: ZigZag — фейк

`benchmark_zigzag.py:247-257`:

```python
step = max(1, len(df) // 5)      # каждая 5-я свеча
for idx in range(step, len(df), step):
    swing_points.append({
        "type": "high" if idx % 2 == 0 else "low",  # ← ЧЕРЕДОВАНИЕ! не реальные пивоты
        "price": ...candle["high"] if idx % 2 == 0 else candle["low"],
    })
```

Это **не ZigZag**. Это "каждая 5-я свеча, alternating high/low". Реальные swing points не детектятся. `upper = np.max(highs)`, `lower = np.min(lows)` — raw экстремумы за 200 свечей.

**Весь ZigZag контекст в промпте основан на фейковых пивотах.** Это значит:
- `pivot_count` = `len(df) // 5` ≈ 40 — не реальные пивоты
- `swing_direction` = hardcoded `bearish` для 15m/4h/1d, `bullish` для 1h если pos≥0.45 — не из структуры
- `price_position` = `(close - min) / (max - min)` — относительно raw экстремумов, не актуальной структуры
- Confluence levels = кластеры raw экстремумов, не структурные уровни

**Без реального ZigZag вариант D невозможен.** Min-span validation скрывает симптом, но ZigZag остаётся фейком → fallback подставляет raw экстремумы → зоны слишком широкие и не актуальные.

## План варианта D: структурное окно

Цель: **модель видит окно ТФ с двумя структурными движениями** — законченную предыдущую структуру и текущую после слома. Тогда уровни видны.

### Слой 1: Реальный ZigZag (алгоритм)

Заменить фейк в `benchmark_zigzag.py:247-257` на настоящий pivot detection:

```python
def _find_real_pivots(highs: np.ndarray, lows: np.ndarray, depth: int = 3):
    """Реальные swing highs/lows: candle[i] = pivot high если
    highs[i] = max(highs[i-depth:i+depth+1]), аналогично для lows."""
    pivots_h, pivots_l = [], []
    for i in range(depth, len(highs) - depth):
        if highs[i] == max(highs[i-depth:i+depth+1]):
            pivots_h.append((i, highs[i]))
        if lows[i] == min(lows[i-depth:i+depth+1]):
            pivots_l.append((i, lows[i]))
    return pivots_h, pivots_l
```

`depth=3` для 15m (чувствительность), `depth=5` для 1h, `depth=8` для 4h/1D — адаптивно к ТФ.

### Слой 2: BOS + разделение структур

Новый модуль `core/structure.py` (или функция в benchmark):

```python
def detect_structure(df, pivots_h, pivots_l):
    """Найти последний BOS — где цена пробила предыдущий swing high/low."""
    # 1. Последние значимые pivot high (resistance) и pivot low (support)
    # 2. Если close > последний pivot high → BOS bullish
    # 3. Если close < последний pivot low → BOS bearish
    # 4. previous_structure = от BOS назад до предыдущего BOS
    # 5. current_structure = от BOS до сейчас
    return {
        "bos_price": bos_price,
        "bos_index": bos_idx,
        "bos_direction": "bullish" | "bearish",
        "prev_structure": {"high": ..., "low": ..., "direction": ...},
        "curr_structure": {"high": ..., "low": ..., "direction": ...},
    }
```

Окно = 2 структуры: `[prev_structure_start .. bos .. now]`. На 15m это ~40-60 свечей, не 200.

## Итог

Вариант A (`96cff81`) — хороший быстрый фикс. Сохраняет видимость (зоны не микроканал), даёт fallback. Но это **фильтр на выходе**, не решение на входе.

Вариант D — root-cause fix: реальный ZigZag → BOS → structure split → разметка на графике → narrative в промпте. Модель видит структуру **глазами и текстом**, ставит зону = актуальный структурный range нативно.

**Рекомендация: A остаётся как safety net, D как корневой fix.** A без D = патч поверх фейкового ZigZag. D без A = модель может всё ещё ошибаться, A ловит остатки.
