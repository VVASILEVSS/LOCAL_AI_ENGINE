## 🔴 Баг HIGH: SL берёт самый дальний swing, а не ближайший

**Файл:** `ollama_client.py:2060`

```python
farther_below = [x for x in below if x < breakout_sl and (current_price - x) <= max_sl_distance]
if farther_below:
    primary["sl"] = farther_below[-1]  # ← БАГ: [-1] = самый дальний!
```

`farther_below` отсортирован по возрастанию (ascending). `[-1]` даёт **самый дальний** swing ниже zone_low. Это значит SL улетает максимально далеко от entry, а не стоит на ближайшем структурном уровне за zone_low.

**Фикс:**
```python
primary["sl"] = farther_below[0]  # ближайший к zone_low (но всё ещё дальше него)
```

Или если нужен ближайший к **entry** (а не к zone_low):
```python
primary["sl"] = farther_below[-1]  # последний в ascending = ближайший к entry
```

Подожди — `below = [x for x in swing_lows if x < current_price]` — это все swing low ниже цены, в порядке их появления в списке. Порядок зависит от `_extract_swing_levels()`. Нужно проверить, какой порядок: хронологический или отсортированный по цене.

**Рекомендация:** добавить логирование `farther_below` перед выбором, чтобы увидеть реальные значения и подтвердить баг.

## 🔴 Баг HIGH: 5M зоны пусто — 3 причины

### Причина 1: Не в дефолтных таймфреймах
`scheduler.py:36` → `_get_timeframes()` возвращает `["15m", "1h", "4h", "1D"]`. 5M нет.

`benchmark_zigzag.py:315` → дефолт `timeframes=["15m", "1h", "4h", "1d"]`. 5M нет.

**Фикс:** Добавить `"5m"` в оба места. Но см. причину 2.

### Причина 2: Окно 50 свечей = 4.2 часа
`benchmark_zigzag.py:16` → `_STRUCT_WINDOW["5m"] = 50`. С ATR distance filter (`_PIVOT_DEPTH=4`, `min_atr_distance=0.5*ATR`), на 50 свечах 5M при низкой волатильности может быть 0-1 пивот. С <2 пивотами → `detect_bos() = None` → zone = `[price, price]` (нулевая ширина).

**Фикс:** Увеличить окно для 5M до 200 (16.7 часов) или 300 (25 часов). Это даст достаточно данных для 4+ пивотов.

### Причина 3: Не в state_tracker
`state_tracker.py:26` → `ACTIVE_REFERENCE_TF_ORDER = ["M15", "H1", "H4", "D1"]`. 5M отсутствует — zone state changes не отслеживаются.

**Фикс:** Добавить `"5M"` в `ACTIVE_REFERENCE_TF_ORDER`.

### Дополнительно: Нет в UI
`handlers.py:78` → `get_tf_keyboard()` перебирает `["15m", "1h", "4h", "1D"]`. 5M нельзя включить через меню бота.
