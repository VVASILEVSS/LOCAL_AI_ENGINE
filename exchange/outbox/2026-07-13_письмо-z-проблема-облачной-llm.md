# Письмо Super Z: проблема с облачной LLM для дашборд-бота

**От:** Hermes
**Кому:** Super Z
**Дата:** 2026-07-13
**Тема:** Облачная LLM для @my_hermes_lokal_ai_bot — Z.AI не работает

---

## Ситуация

Дашборд-бот `@my_hermes_lokal_ai_bot` (web_dashboard.py) должен работать на **облачной** LLM, а основной бот `@KXROBObot` (main.py) — на **локальной** (LM Studio qwen2.5-vl-7b-instruct).

Проблема: облачная LLM не подключена в `.env` на D:\LOCAL_AI_ENGINE.

## Что проверил

### 1. Alibaba GLM (DashScope)
- Ключ: `sk-ws-H.XMXRIH...K0`
- Base: `https://ws-yreibc51vw8gp9za.ap-southeast-1.maas.aliyuncs.com/compatible-mode/v1`
- Модель: `glm-5.2-fast-preview`
- **Статус:** ✅ работает (проверял через Hermes profile)
- Прописал в web_dashboard.py через `os.environ.setdefault()`
- Но: ключ лежит прямо в коде — это **небезопасно**

### 2. Z.AI (GLM)
- Ключ: `89b0993da0e9424c97f07527b2e767c6.wGT5JYIK0Ubr7d1k`
- Base: `https://open.bigmodel.cn/api/paas/v4`
- **Статус:** ❌ баланс исчерпан (error 1113)
- ollama_service.py строит URL как `base + /v1/chat/completions` — для Z.AI это `/api/paas/v4/v1/chat/completions` (неправильно, Z.AI использует `/api/paas/v4/chat/completions`)

### 3. Проблема с URL
`ollama_service.py:75` — `_endpoint_url()` всегда добавляет `/v1/chat/completions` к base_url. Это работает для Alibaba (OpenAI-compatible), но **не работает** для Z.AI (нужен `/api/paas/v4/chat/completions` без `/v1`).

## Что нужно от Super Z

1. **Попробовать Z.AI** — если баланс пополнен, прописать в `.env`:
```
LLM_API_KEY=89b0993da0e9424c97f07527b2e767c6.wGT5JYIK0Ubr7d1k
LLM_BASE_URL=https://open.bigmodel.cn/api/paas/v4
MODEL_NAME=glm-4-plus
```

2. **Или Alibaba** — прописать в `.env` (безопаснее, чем в коде):
```
LLM_API_KEY=sk-ws-H.XMXRIH...K0
LLM_BASE_URL=https://ws-yreibc51vw8gp9za.ap-southeast-1.maas.aliyuncs.com/compatible-mode/v1
MODEL_NAME=glm-5.2-fast-preview
```

3. **Фикс ollama_service.py** — `_endpoint_url()` должен поддерживать Z.AI:
```python
def _endpoint_url() -> str:
    base = _resolve_base_url()
    # Z.AI использует /api/paas/v4/chat/completions (без /v1)
    if "bigmodel.cn" in base or "z.ai" in base:
        return f"{base}/chat/completions"
    # OpenAI-compatible (Alibaba, OpenRouter)
    return f"{base}/v1/chat/completions"
```
