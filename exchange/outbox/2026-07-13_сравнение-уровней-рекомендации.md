## Рекомендации для Super Z

### 1. Веб-дашборд: убрать автозапуск бота
**Проблема:** `web_dashboard.py` запускает второго бота в фоновом потоке → `TelegramConflictError` (двойной getUpdates).
**Решение:** Убрать `start_bot_thread()` из `__main__`. Веб-дашборд должен только читать из БД, не запускать бота.

```python
if __name__ == "__main__":
    # start_bot_thread() — УБРАТЬ
    app.run(host="0.0.0.0", port=WEB_PORT, debug=False)
```

### 2. Сохранять ВСЕ прогнозы в signal_log
**Проблема:** `save_signal_log()` вызывается только при active signals (aggressive_breakout, retest, reversal, false_breakout). При no_signal прогноз не сохраняется → нет данных для backtest статистики.
**Решение:** Сохранять все прогнозы, независимо от signal_status. Это даст:
- Полный backtest статистику
- Веб-дашборд покажет данные даже при no_signal
- A/B тест можно сравнивать даже без активных сигналов

### 3. key_zones: LLM не всегда выводит
**Проблема:** В анализе BTC (22:34) LLM не вывел `key_zones` (resistance=null, support=null), хотя в ETH (23:36) вывел корректно.
**Решение:** Добавить fallback в `enforce_risk_rules` — если LLM не вывел key_zones, взять из confluence_levels (первый resistance + первый support).

### 4. tf_span_map: расхождение между прогонами
**Проблема:** tf_span_map 4h: 1366.3 (run 1) vs 1376.3 (run 2). Это вычисляется из tf_zones, которые идентичны → span должен быть одинаковым.
**Решение:** Проверить формулу расчёта tf_span_map. Возможно, округление или разные данные (A TR vs raw high-low).

### 5. ThreadedResolver фикс закоммитить
**Проблема:** main.py на D:\LOCAL_AI_ENGINE имеет локальный фикс (ThreadedResolver monkey-patch), но не закоммичен в git.
**Решение:** Закоммитить фикс в main.py, чтобы при следующем git pull он не потерялся.

### 6. confluence_levels: run 2 добавляет второй уровень
**Проблема:** При temp=0.25 LLM добавил второй confluence level (1832.17), при temp=0.15 — только один.
**Решение:** Это нормальное поведение (более высокая temp → больше креативности). Не фиксить, но учесть при анализе результатов self-consistency.

### 7. Веб-дашборд: добавить last_analysis_time
**Проблема:** В БД нет трекинга последнего анализа. Веб-дашборд показывает "—".
**Решение:** Добавить в settings таблицу `last_analysis_time` key, обновлять при каждом цикле scheduler.
