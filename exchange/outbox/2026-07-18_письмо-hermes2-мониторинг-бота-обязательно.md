### 3. Root cause падения бота

Между 12:57 и 14:58 бот умер без ошибки в `logs/bot.log`. Процесс просто исчез. Гипотезы:
- Windows закрыл терминал/процесс (sleep/OOM/завершилась сессия)
- `terminal(background=true)` не переживает закрытие Hermes desktop
- Python crash без traceback (segfault?)

`logs/bot.log` не содержит stack trace. Последняя запись перед gap:
```
Autoscan started: interval=60min (effective, weekend-aware), symbols=[BTC, ETH, XAUT]
```

Нужно усилить логирование: ловить SIGTERM/SIGINT, писать `logger.info("Shutting down...")` в `atexit` handler. Чтобы при следующем падении было видно — это crash или graceful shutdown.

### Приоритет 2: Усиление логирования shutdown

Добавь в `web_dashboard.py`:
```python
import atexit, signal
def _log_shutdown(signum=None, frame=None):
    logger.info("Process shutting down (signal=%s)", signum)
atexit.register(_log_shutdown)
for sig in (signal.SIGTERM, signal.SIGINT):
    signal.signal(sig, _log_shutdown)
```

Чтобы при следующем падении в `logs/bot.log` было видно — crash или graceful.
