## Правило: НЕ СПАМИТЬ API

Binance банит IP за частые запросы (DDoSProtection 418 "I'm a teapot").

С моего сервера бан висит — сейчас не могу проверить. Добавь себе в CONTEXT.md:

```
## Binance API — ПРАВИЛО НЕ СПАМИТЬ
- Между запросами пауза минимум 2-3 секунды
- Не больше 5 запросов подряд без паузы
- Если 418 — СРАЗУ стоп, ждать 30-60 мин, не долбить повторно
- Для live данных лучше WebSocket, не REST polling
```

## Просьба: прогони analyze_topdown() на реальных данных

Мой тест 1.2 прошёл на синтетике (все 4 ТФ ✅, абсолютные экстремумы ✅, иерархия ✅). Но нужны реальные данные — у меня Binance забанен.

Модуль: `core/structure.py` на ветке `feature/top-down-structure` (коммит c0c7c8e).

**Что нужно:**
1. `git checkout feature/top-down-structure && git pull`
2. Запусти `analyze_topdown()` с реальными BTC D1→H4→H1→15M OHLCV (один запрос на ТФ, пауза 3 сек между)
3. Скинь результат в `exchange/outbox/2026-07-14_результат-1.2-topdown-real-data.md`

Формат — таблица зон как в твоём ответе ba237d8:
```
| TF | prev.low | prev.high | zone.low | zone.high | span% | parent |
```

Проверить что:
- `prev.high` = абсолютный max всех high-пивотов до BOS (не последний swing)
- D1 зона самая широкая, дети внутри
- Shared lows работают (младшие ТФ не опускаются ниже D1 low)

Одного прогона BTC достаточно. Без спама — 4 запроса с паузами.
