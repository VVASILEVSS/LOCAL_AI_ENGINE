
| Зона = recent 40% пивотов (статистика) | Зона = крайние swing H/L последней субструктуры после BOS (структура) |

| Нет концепции целей | Цели = границы старшего ТФ + значимые swing levels |
| Объём = volume_ratio метрика | Объём = фильтр у уровня: пробой или отскок |

## Концепция: Top-Down Structural Pipeline

### Принцип 1: Анализ сверху вниз

```
D1 → prev_structure (начало-конец-субструктуры) → фиксируем
D1 → last BOS → curr_structure
D1 → крайние swing H/L последней субструктуры = ГЛОБАЛЬНЫЕ ГРАНИЦЫ
     ↓ эти границы обрамляют H4
H4 → внутри D1 range → BOS → крайние свинги = H4 границы
     ↓
H1 → внутри H4 range → BOS → крайние свинги = H1 границы
     ↓
M15 → внутри H1 range → BOS → крайние свинги = M15 границы
```

Каждый ТФ анализируется **в рамках границ старшего ТФ**. Младший ТФ не может выйти за границы старшего (если вышел = был BOS на старшем, нужно пересмотреть).

### Принцип 2: Зона = структурный range после BOS

Зона ТФ = **крайние swing high и swing low последней субструктуры после BOS**.

- Не "recent 40% пивотов"
- Не "N последних свечей"
- Не raw max/min всей выборки
- **Структура**: где цена остановилась (swing), где началась последняя субструктура

Ширина зоны варьируется — сильное движение = шире, короткое = уже. **Без привязки к кол-ву свечей.**

### Принцип 3: Общие границы = confluence

Если уровень держит цену, несколько ТФ могут иметь общие границы. Это **нормально** — не нужно раздвигать (`_enforce_zone_uniqueness` делает наоборот).

- Общий уровень = **confluence** (сильный уровень)
- Может быть confluence всех 4 ТФ (если уровень не пробивается)
- Другая сторона полигоно имеет структуру (граница)

### Принцип 4: Накопление = структурное определение

**Накопление для каждого ТФ = цена внутри зоны своего ТФ + нет обновления HH/LL.**

Не market_mode метрика, не price_position — **структурное отсутствие новых экстремумов**.

```
Если цена внутри зоны ТФ
И последние N пивотов не делают новых HH / новых LL относительно зоны
→ накопление на этом ТФ
```

### Принцип 5: Цели при выходе из накопления

При выходе из накопления цена стремится к:
1. **Границам старшего ТФ** (следующий уровень структуры)
2. **Значимым уровням субструктуры** (swing levels внутри структуры)
3. **Значимым уровням структуры** (крайние свинги старшей структуры)

Это и есть **цели** — куда пойдёт цена после выхода из накопления.

### Принцип 6: Объём = фильтр у уровня

При подходе цены к уровню:
- **Объём растёт** → реальный пробой (цена пойдёт дальше)
- **Объём падает** → отскок (ложный вынос)

Не абстрактный volume_ratio, а **объём в момент подхода к конкретному swing level**.

## Что меняется архитектурно

### Сейчас (независимый анализ)

```python
for tf in timeframes:           # D1, H4, H1, M15 — каждый сам по себе
    pivots = _find_real_pivots(df, depth)
    upper = max(recent_40% pivots)
    lower = min(recent_40% pivots)
    # → LLM получает zone, копирует
```

### Нужно (top-down chain)

```python
d1_analysis = analyze_tf_structure(d1_pivots, d1_bos, d1_price)
d1_zone = (d1_analysis.zone_low, d1_analysis.zone_high)  # ГЛОБАЛЬНЫЕ ГРАНИЦЫ

h4_analysis = analyze_tf_structure(h4_pivots, h4_bos, h4_price, parent_zone=d1_zone)
# H4 зона внутри D1 range
h4_zone = (h4_analysis.zone_low, h4_analysis.zone_high)

h1_analysis = analyze_tf_structure(h1_pivots, h1_bos, h1_price, parent_zone=h4_zone)
# H1 зона внутри H4 range

m15_analysis = analyze_tf_structure(m15_pivots, m15_bos, m15_price, parent_zone=h1_zone)
# M15 зона внутри H1 range
```

`analyze_tf_structure` принимает `parent_zone` — рамку от старшего ТФ. Зона младшего ТФ **не может** выйти за parent_zone (если вышла = parent BOS сломан).
