## Что создано

### 1. `core/ollama_service.py` (NEW, 237 строк)

Единый транспортный модуль для вызова LLM. Не использует библиотеку `ollama` — только `httpx`.

**Функция `generate()`:**
```python
async def generate(
    messages: list[dict],
    *,
    model: str = "",
    temperature: float = 0.02,
    max_tokens: int = 1200,
    timeout: int = 30,        # спецификация задачи
    retry: bool = True,
) -> dict:
    # Returns: {"content": str, "model": str, "usage": dict, "raw": dict}
    # Raises:  LLMError on persistent failure
```

**Два режима (автодетект через `config.LLM_MODE`):**
| Режим | Условие | Authorization header | base_url |
|-------|---------|---------------------|----------|
| `cloud` | `LLM_API_KEY` непустой | `Bearer <key>` | из `LLM_BASE_URL` |
| `local` | `LLM_API_KEY` пустой | нет | из `LOCAL_AI_ENDPOINT` (derived) |

**Retry:** 3 попытки с exponential backoff (1s → 2s → 4s). Retry на 5xx, 429, ReadTimeout, ConnectError. Не retry на 4xx (кроме 429).

**`LLMError` exception** — для всех transport failures (timeout, 5xx, network, no key in cloud mode). Атрибуты: `status`, `attempt`.

**Доп.:**
- `health_check()` — GET `/v1/models`, возвращает bool
- `info()` — конфиг для логирования (без ключа)

```python

    mode_hint = "cloud" if LLM_MODE == "cloud" else "local (LM Studio)"
    return {"error": True, "message": f"LLM ({mode_hint}): {e}"}
except Exception as e:
    logger.exception("Ошибка запроса")
    return {"error": True, "message": f"Неожиданная ошибка LLM: {type(e).__name__}: {e}"}
```

## Важно: баг `/v1/v1/`

`LLM_BASE_URL` для cloud должен быть **БЕЗ** `/v1` суффикса — `ollama_service.generate()` сам добавляет `/v1/chat/completions`. Для Alibaba endpoint:
- ❌ `https://...com/compatible-mode/v1` → `/v1/v1/chat/completions` → 404
- ✅ `https://...com/compatible-mode` → `/v1/chat/completions` → 200

Для local mode (`http://localhost:1234`) это неактуально — нет `/v1` в базе.
