## Баг №1: Direction не пробрасывается назад (КРИТИЧНО)

**Файл:** `core/ollama_client.py`, `enforce_risk_rules()` ~L1811

**Что происходит:**
- V4 `enforce_risk_rules` вычисляет `direction_hint` через `_direction_from_data(data)` (по `current_substructure`: `breakout_up` → long, `breakout_down` → short)
- V4 считает SL/TP для правильного direction
- Но **НЕ ЗАПИСЫВАЕТ** `direction_hint` обратно в `data["signal_direction"]`
- `position_tracker` (scheduler L642) читает `parsed.get("signal_direction")` — **пусто** (LLM не вернула поле)
- Fallback: position_tracker берёт direction из `ltf_structure` → часто **противоположный**
- → V4 посчитала SL/TP для long, position_tracker открыл **short** → **SL/TP инвертированы**

**Пример (signal_log id=760, позиция 7):**
```
direction=short, entry=66368
sl=66153 (НИЖЕ entry!) ← для short должно быть ВЫШЕ
tp1=66797 (ВЫШЕ entry!) ← для short должно быть НИЖЕ
```
V4 посчитала для long (SL ниже, TP выше), трекер открыл short → инверсия.

**Фикс** (commit `95c5b05`):
```python
direction_hint = _direction_from_data(data)
data["signal_direction"] = direction_hint  # ← ДОБАВЛЕНО
```

**В runtime проверено:** `dir=long` в логе (раньше `dir=` пустой).

## Баг №2: Reverse на любой opposite-direction сигнал (КРИТИЧНО)

**Файл:** `core/position_tracker.py`, `handle_new_signal()` ~L339-348

**Что происходит:**
- LLM меняет long↔short каждые 15-45 мин (сигналы `aggressive_breakout` с разным `current_substructure`)
- position_tracker реверсирует на **КАЖЫЙ** opposite-direction сигнал — без guard-условий
- 4 из 5 reverse-закрытий пришлись в минусе (rr<0.5): цена не дошла до TP1, а трекер уже реверсировал
- Только 1-я позиция (hold 37h) закрылась в плюс — потому что цена ушла далеко за TP1

**Паттерн потерь:**
| ID | Direction | Entry→Close | RR | Hold | Result |
|----|-----------|-------------|-----|------|--------|
| 1 | long | 64871→66150 | 3.35 | 37h | ✅ WIN |
| 3 | short | 66150→66204 | 0.40 | 45m | ❌ LOSS |
| 4 | long | 66204→66346 | 0.25 | 7h | ❌ LOSS |
| 5 | short | 66346→66446 | 0.20 | 45m | ❌ LOSS |
| 6 | long | 66446→66368 | 0.27 | 15m | ❌ LOSS |

**Фикс** (commit `95c5b05`):
```python
# Reverse только если:
#   (a) цена дошла до TP1 (in profit) — reverse фиксирует прибыль
#   (b) позиция открыта ≥90 мин — тренд сменился по-настоящему
# Иначе — ignore, ждём SL/TP
MIN_HOLD_MINUTES = 90

if not in_profit and hold_minutes < MIN_HOLD_MINUTES:
    logger.info("POSITION reverse-blocked ...")
    return "ignored"
```

**В runtime проверено:** `action=ignored` (same direction) — раньше был бы `reversed`.

## Баг №3: Нет safety-net проверки SL/TP orientation

**Файл:** `core/position_tracker.py`, ~L321 (после `if not (entry and sl and tp1)`)

**Что происходит:**
- Если direction mismatch (баг №1), position_tracker всё равно открывал позицию с инвертированными SL/TP
- L2263-2276 в `enforce_risk_rules` обнуляет invalid SL/TP, но не всегда срабатывает (edge cases с BOS broken_level)

**Фикс** (commit `95c5b05`):
```python
# long: SL < entry, TP1 > entry | short: SL > entry, TP1 < entry
if direction == "long":
    if sl_f >= entry_f or tp1_f <= entry_f:
        return "skipped"
elif direction == "short":
    if sl_f <= entry_f or tp1_f >= entry_f:
        return "skipped"
```
