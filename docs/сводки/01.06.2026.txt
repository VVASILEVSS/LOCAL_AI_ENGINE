## 8) Что показали скрины Pine-настроек

### Рабочие параметры BTC, найденные экспериментально:
#### Preset / TF
- `P2 TF = 60`
- `P2 Pivot Left = 9`
- `P2 Pivot Right = 9`
- `P2 Min Price % = 0.0075`
- `P2 Min Flow % = 0.004`
- `P2 Min ATR mult = 0.4`
- `P2 Cooldown = 10`

#### Regime
- `Regime Lookback = 11`
- `Min EMA Slope % = 0.08`
- `Min EMA Range % = 0.12`
- `Regime Confirm Bars = 3`

#### Bias Weights
- `Weight: Flow Slope = 60`
- `Weight: CMF Score = 25`
- `Weight: Regime = 15`
- `Bias Threshold = 2`

### Вывод:
Это **эталонные рабочие значения для BTC**, и они показали, что:
- Python candidate thresholds были слишком жёсткими
- autotune не должен ломать рабочий диапазон
- нужен адаптивный auttune под инструмент

---

## 9) Что мы решили по autotune стратегии
### Правильный подход:
- не ручная подкрутка под каждый тикер
- не 1:1 жёсткая копия Pine
- а **инструмент-адаптивный autotune**

### Он должен учитывать:
- волатильность
- объём
- шумность структуры
- частоту pivot-ов
- класс инструмента:
  - BTC-like
  - ETH-like
  - XAU-like
  - default
