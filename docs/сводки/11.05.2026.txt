# 1) Цель проекта сейчас

Проект `D:\telega\LOCAL_AI_ENGINE` — это система анализа рынка для Telegram/LLM, которая должна:

- получать данные по нескольким инструментам
- анализировать структуру на нескольких ТФ
- определять режим рынка:
  - balance
  - trend
  - correction
  - breakout / false breakout
- строить зоны
- готовить понятный текстовый вывод в Telegram
- отдавать компактный контекст в LM Studio / LLM
- позже использовать ZigZag как дополнительный экстремум-фильтр
- позже добавить liquidity heatmap

# 2) Текущая концепция пайплайна

## Сейчас
У тебя уже есть **основная логика анализа**, которая формирует такие сообщения:

- HTF структура
- LTF структура
- накопление/распределение
- волновая фаза
- ABC риск
- зоны
- зоны по ТФ
- signal / aggressive / conservative
- risk / R:R
- confidence

Это значит, что **основной engine уже работает как market-state classifier**, а не просто как ZigZag-детектор.

## ZigZag сейчас
ZigZag:
- не является главным источником истины
- тестируется отдельно
- используется для поиска экстремумов и сравнения с визуальной структурой
- нужен как потенциальный улучшатель признаков

# 3) Полная архитектура проекта

## A. Data acquisition layer
Отвечает за:
- загрузку OHLCV
- получение объёмов
- исторические данные по нескольким инструментам и ТФ

### Вероятные модули:
- `binance_metrics.py`
- возможно часть `main.py`
- `db.py`

### Функции:
- fetch market data
- cache/store data
- normalise to DataFrame
- prepare multi-TF windows

---

## B. Core market structure layer
Это главный слой проекта.

### Что он делает
- определяет HTF/LTF структуру
- классифицирует рынок:
  - Balance
  - Trend
  - Correction
  - Breakout
  - False breakout
- определяет ключевые зоны
- оценивает волновую фазу
- оценивает ABC риск
- строит сигналы

### Главный смысл
Этот слой — **основной аналитический мозг**, а ZigZag — только помощник.

### Вероятные файлы
- `auto_chart.py`
- `handlers.py`
- `main.py`
- `config.py`
- `utils.py`

---

## C. ZigZag experiment layer
Отдельный экспериментальный слой.

### Назначение
- найти экс  ремумы
- сравнить с реальным графиком
- проверить пригодность для ML / LM Studio
- убедиться, что логика совпадает с TradingView / Binance
- подготовить альтернативную основу, если текущая структура недостаточно хороша

### Файлы
- `core/structural_zigzag.py`
- `core/zigzag_forecast_test.py`
- `core/structure_levels_test.py`

### Статус
**Не включён в основной проект как truth-source.**  
Сейчас это отдельный research/test module.

---

## D. Telegram messaging layer
Это слой, который превращает аналитику в читаемый телеграм-формат.

### Что он делает
- формирует заголовок
- показывает цену
- выводит HTF/LTF структуру
- показывает зоны
- показывает сигнал
- показывает риск/TP/SL
- добавляет пояснения
- делает confidence summary

### Результат
Именно сюда должны попадать:
- структура
- зигзаговые экстремумы
- зоны
- паттерны
- liquidity later

---

## E. LLM / LM Studio layer
Используется для:
- компактного summary
- reasoning
- классификации режимов рынка
- объяснений для Telegram
- позже — для ML assistance

### Сейчас
- ZigZag тестируется отдельно
- потом будет использоваться как feature source для LLM

# 4) Как сейчас должны сосуществовать основные и zigzag-слои

## Основной слой
Он должен отвечать на вопрос:
> “Что происходит на рынке прямо сейчас?”

### Пример:
- Balance
- Correction down
- False breakout
- Trend continuation
- Risk of ABC

## ZigZag слой
Он должен отвечать на вопрос:
> “Где реальные экстремумы и swing-движения?”

### Пример:
- pivot high
- pivot low
- swing direction
- amplitude
- local reversal logic

# 7) Как это связано с ZigZag

ZigZag может помочь в нескольких местах:

### Для структуры
- подтвердить swing direction
- подтвердить локальные экстремумы

### Для зон
- помочь уточнить ключевые highs/lows

### Для wave phase
- показать завершённость импульса

### Для ABC risk
- найти предпосылки для коррекции

Но ZigZag не должен:
- заменять HTF/LTF classifier
- ломать existing market regime logic
- быть единственным источником сигнала
