## 4. ГЛАВНЫЙ ИНСАЙТ: «слом тенденции» ≠ «слом структуры»

Возный **явно разделяет** два понятия:

> «Когда у нас происходит **слом структуры, слом тенденции без пробоя трендовых линий**»

- **Слом структуры (BOS)** = пробой high/low субструктуры → структура сломана
- **Слом тенденции** = пробой **трендовой линии** → тренд сломан
- Они могут происходить **не одновременно**. Структура ломается раньше, трендовая линия позже.

Это **точно** наш T10-T11 спек:
- T10 = `detect_trend_lines(pivots)` — рисует трендовые по субструктурам
- T11 = `check_line_break()` + false breakout — определяет истинность пробоя

Возный подтверждает подход: **зона = где цена, трендовая линия = куда идёт, угол = импульс vs коррекция** — это и есть «третье измерение».

## 5. Методология Возного (по шагам)

### Шаг 1 — D1 (долгосрок)
- Ищет «свечу среды» — её high/low = **зона накопления**
- Строит max/min предыдущего движения = среднесрочные цели
- «Чем дольше накопление — тем сильнее движение»

### Шаг 2 — H4 / H1 (среднесрок)
- Строит **субструктуры** — ломаную линию через каждую точку разворота
- Проверяет условие тренда: «есть пробой low, нет пробоя high» (нисходящий)
- **Слом структуры** = пробой high субструктуры меняет тренд

### Шаг 3 — H1 → M15 (top-down nesting)
- На H1 строит зигзаги → каждая точка разворота = **уровень/цель**
- «Зеркальный уровень» = старый pivot, к которому цена вернётся
- Переходит на M15 → ищет точку входа

### Шаг 4 — Вход
- **Пробой уровня** = «торгуем с крупными участниками, толпа уровни не пробивает»
- **Отбой уровня** = торгуем к **центру** ценового диапазона (50% = баланс)
- Стоп = над противоположным high/low проторговки
- Цели: 50% → 100% высоты предыдущего движения

## 6. Что у Возного ЕСТЬ, а у нас НЕТ (кандидаты на Roadmap)

1. **Fibonacci 50% / 61.8% / 100% от высоты тренда** — встроенные цели коррекции
   - «50% — всегда самая серьёзная цель»
   - «без пробоя 61.8% не будет поломки тренда»
   - Кандидат: **T14: `compute_fib_targets()`** (отложено пользователем)

2. **CME Open Interest** — крупные контракты как магнит цены
   - Уровни где 370 контрактов на продажу vs 33 на покупку
   - Это не volume filter, это **order flow** — серьёзно сложнее
   - Кандидат: **T6+ (расширение)**

3. **Свеча как зона накопления** — паттерн: «N свечей D1 внутри предыдущей»
   - Можно детектировать программно

4. **Зеркальный уровень** — старый pivot становится целью
   - У нас `targets = parent boundaries`, но Возный явно ищет **отдельные pivot-точки**
